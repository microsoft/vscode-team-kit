description: Fix a github issue
---

The user will provide you with a Github issue to fix

- Discover the user's Github handle/name
- Read and understand the Github issue and any related discussion.
- Make a branch `<github-name>/<issue-number>`
- Understand existing code patterns and identify how the fix should be made in a pragmatic way that uses appropriate abstractions and reduces duplication
- Implement your fix
- Add or modify tests to validate what you did and use the runTests tool to ensure they pass
- Provide a brief summary of what you changed, and then stop
